---
name: Map marker → GPU layer migration pattern
description: How map.tsx renders high-count map points via GeoJSONSource layers instead of native Markers, and the MapLibre RN v11 facts that make it work
---

Rule: high-count map point categories (trail dots, agency pins, campgrounds, hike POIs, OSM flags) must be GeoJSONSource + CircleLayer (+ SymbolLayer for glyphs), never per-point native `<Marker>` views. Native Markers are reserved for low-count/animated items (user pulse, SOS, crew, keypoints, community notes).

**Why:** ~500+ native Marker views made pan/zoom visibly sluggish; GPU layers render in the map engine with no per-view React/native overhead.

**How to apply:**
- Feature `properties` carry an `id` (or index `i`); a memo returns `{ collection, byId|items }` so the press handler can look up the full item. Handlers must call `e.stopPropagation()` or the press bubbles to the Map.
- MapLibre RN v11 press facts (verified in library source): default hitbox is 44×44; when sources overlap, only the top-most layer's source receives the press — no double-fire.
- Glyphs: PNGs generated by `scripts/src/map-icons/generate.ts` (sharp + @mdi/js, scripts-only devDeps), registered via `<Images>` as a direct Map child. require()'d single-density PNG = scale 1, so rendered dp = png px × icon-size (52px × 0.25 = 13dp). Use `icon-allow-overlap` + `icon-ignore-placement`.
- `<Images>` survives style swaps because every style change remounts the whole MapView (key prop); no re-registration handling needed.
- Every source stays gated on `mapStyleLoaded` (Android silently drops sources registered before style load; the flag is reset on both style-swap paths). Pins briefly vanish during a style swap until `onDidFinishLoadingStyle` — expected, not a bug.
- White glyph on colored circle is the norm; invert (colored glyph on white circle, e.g. checkered flags) by pre-coloring the PNG.
